# Changelog — 2026-03-24

## Tiered Memory System

Added a persistent, multi-tier memory system for the financial AI agent. Memories are stored in SQLite and injected into every agent session, reducing token usage and enabling the agent to retain context across conversations and across multiple users.

### Architecture

Three memory tiers:

| Tier | Scope | TTL | Purpose |
|------|-------|-----|---------|
| `user_memories` | Per sender JID | None | User profile (risk tolerance, goals, preferences) |
| `topic_memories` | Per topic + group | 7 days | Analysis summaries (company research, sector views) |
| `global_facts` | All users | None | Shared macro data (interest rates, commodity prices) |

### How it works

**Injection**: At trigger time, relevant memories are fetched from SQLite and prepended to the agent prompt as an XML `<memory>` block:

```xml
<memory>
<global_facts>
  <fact key="singapore_tbill_rate">3.5 to 4.0% as of March 2026</fact>
</global_facts>
<user_profile>
  <fact key="risk_tolerance">conservative</fact>
  <fact key="investment_focus">dividend stocks in Singapore</fact>
</user_profile>
<topic_cache>
  <topic name="SATS Ltd" updated="...">Dividend yield 1.35-1.62%, below T-bill rate...</topic>
</topic_cache>
</memory>
```

**Extraction**: After each completed agent turn, a Haiku call asynchronously extracts new facts from the conversation and upserts them into the appropriate tier. This is fire-and-forget — it never delays user responses.

### Files changed

- **`src/db.ts`** — Added 3 new SQLite tables (`user_memories`, `topic_memories`, `global_facts`) and CRUD functions (`getUserMemories`, `upsertUserMemory`, `getTopicMemories`, `upsertTopicMemory`, `pruneExpiredTopicMemories`, `getAllGlobalFacts`, `upsertGlobalFact`)
- **`src/router.ts`** — Added `MemoryContext` interface and `formatMessagesWithMemories()` which prepends the `<memory>` XML block to the prompt
- **`src/memory-extractor.ts`** — New file. Async Haiku-based extraction that reads `ANTHROPIC_API_KEY` or `CLAUDE_CODE_OAUTH_TOKEN` from `.env` and calls `api.anthropic.com` directly
- **`src/index.ts`** — Memory injection at both prompt-build sites in `processGroupMessages` and `startMessageLoop`; extraction triggered on `result.status === 'success'` in the streaming callback; daily prune of expired topic memories on startup
